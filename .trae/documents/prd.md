## 1. 产品概述

迷宫闯关小游戏——一款浏览器端随机迷宫闯关游戏，玩家使用方向键控制角色穿越随机生成的迷宫，收集钥匙、躲避巡逻敌人、拾取道具，逐关挑战递增难度。目标用户为休闲游戏玩家，核心价值在于策略性的路线规划与紧张刺激的躲避体验。

## 2. 核心功能

### 2.2 功能模块

1. **开始界面**：游戏标题、开始按钮、操作说明
2. **游戏界面**：迷宫地图、角色控制、HUD信息栏
3. **通关提示**：关卡完成动画、得分统计、下一关按钮
4. **失败提示**：失败动画、重试按钮

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 开始界面 | 标题区域 | 游戏Logo与标题动画展示 |
| 开始界面 | 操作说明 | 方向键移动、规则说明 |
| 开始界面 | 开始按钮 | 点击进入第一关 |
| 游戏界面 | 迷宫画布 | Canvas渲染迷宫、角色、敌人、道具、钥匙 |
| 游戏界面 | HUD信息栏 | 当前关卡、得分、剩余钥匙数、道具状态、倍率 |
| 游戏界面 | 道具计时器 | 显示加速/无敌剩余时间与叠加层数 |
| 通关提示 | 得分展示 | 本关得分与累计得分 |
| 通关提示 | 下一关按钮 | 进入下一关 |
| 失败提示 | 失败原因 | 被敌人碰到 |
| 失败提示 | 重试按钮 | 重新开始本关 |

## 3. 核心流程

玩家从开始界面进入游戏，每关随机生成迷宫。玩家用方向键移动角色，需收集足够数量的钥匙才能打开出口门。迷宫中有巡逻敌人沿固定路线来回移动，碰到敌人则本关失败重新开始。地图上随机刷新加速和无敌道具，拾取后获得临时增益效果。连续拾取道具不被敌人碰到则得分倍率递增，被碰到则倍率重置为1。收集所有钥匙后走到出口门即可通关，进入下一关，迷宫尺寸和敌人数量递增，至少5关。

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：暗色系（深蓝黑 #0a0e1a），霓虹风格强调色（青色 #00f0ff、品红 #ff0066、金色 #ffd700）
- 按钮：圆角霓虹发光风格，hover时脉冲发光
- 字体：像素风/科技感字体，标题用 Press Start 2P，正文用 Orbitron
- 布局：居中Canvas，顶部HUD信息栏，暗色氛围背景
- 图标：像素风图标，钥匙、闪电（加速）、盾牌（无敌）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 开始界面 | 标题区域 | 像素风大标题、霓虹发光动画、深色背景粒子效果 |
| 开始界面 | 操作说明 | 半透明卡片、像素图标+文字说明 |
| 开始界面 | 开始按钮 | 霓虹边框大按钮、hover脉冲动画 |
| 游戏界面 | 迷宫画布 | Canvas居中、像素风渲染、深色迷宫墙壁+霓虹边框 |
| 游戏界面 | HUD信息栏 | 顶部栏、关卡/得分/钥匙/道具状态/倍率 |
| 通关提示 | 弹窗覆盖 | 半透明遮罩+居中卡片、金色光芒动画、得分数字 |
| 失败提示 | 弹窗覆盖 | 半透明红色遮罩+居中卡片、重试按钮 |

### 4.3 响应式

- 桌面端优先，Canvas自适应窗口大小
- 移动端：虚拟方向键触控
- 触摸优化：大尺寸虚拟按键

### 4.4 游戏元素视觉

- 角色：青色发光方块/小人
- 墙壁：深色+霓虹边线
- 钥匙：金色旋转钥匙图标
- 敌人：红色闪烁方块/幽灵
- 加速道具：蓝色闪电图标
- 无敌道具：金色盾牌图标
- 出口门：未开时灰色锁定，钥匙足够后绿色发光
